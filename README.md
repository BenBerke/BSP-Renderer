# BSP-Based Sector Engine

A retro-style 2.5D software rendering engine written in C++ that uses a **Binary Space Partitioning (BSP) Tree** for spatial partitioning, visibility ordering, and sector lookup.

The project demonstrates how classic engines such as Doom and Build-style renderers organize world geometry into a BSP structure while supporting sectors with different floor and ceiling heights.

---

## Overview

The world is constructed from:

* **Walls** represented as line segments
* **Sectors** containing floor and ceiling heights
* **Portal walls** connecting neighboring sectors
* A **BSP tree** generated automatically from wall geometry
* **Sub-sectors** generated by clipping sector polygons through BSP planes

The BSP serves two primary purposes:

1. **Rendering Order**

   * Produces a back-to-front wall traversal without requiring depth buffering.
   * Walls are visited in painter's-order relative to the player's position.

2. **Spatial Queries**

   * Quickly determines which sector the player occupies.
   * Converts arbitrary sector polygons into BSP leaf nodes (sub-sectors).

---

# BSP Construction

## Splitter Selection

Every BSP node stores a wall that acts as the partitioning plane.

Candidate walls are scored using:

```cpp
abs(frontCount - backCount) + spanningCount * 8
```

The heuristic attempts to:

* Keep the tree balanced
* Minimize wall splits
* Reduce recursion depth

Walls that cross the splitter are heavily penalized because splitting geometry increases tree complexity.

---

## Wall Classification

Each wall is classified against the splitter:

```text
Front
Back
Coplanar
Spanning
```

Classification is performed using the signed distance of both wall endpoints to the splitter line.

```cpp
PointSide(splitter, point)
```

Internally this uses a 2D cross product:

```cpp
splitter.dir.Cross(point - splitter.start)
```

Positive values lie in front of the splitter, negative values behind it.

---

## Wall Splitting

When a wall spans both sides of a partition:

```text
A --------X-------- B
```

it is divided into two new walls:

```text
A --------X
X-------- B
```

The intersection point is computed analytically and each fragment is inserted into the appropriate child node.

This guarantees that every wall contained within a BSP node lies entirely on one side of the partition.

---

## Recursive Tree Generation

The BSP is built recursively:

```text
Choose Splitter
│
├── Front Walls
│   └── Build Front Subtree
│
└── Back Walls
    └── Build Back Subtree
```

Leaf nodes are generated when no walls remain on a side.

Each internal node stores:

* Splitter wall
* Coplanar walls
* Front child
* Back child

---

# Sub-Sector Generation

A BSP by itself partitions walls, but sectors are polygonal regions.

To associate BSP leaves with sectors:

1. Each sector polygon is supplied to the BSP.
2. The polygon is recursively clipped against every splitter.
3. Polygon fragments are propagated into child nodes.
4. When a leaf is reached, the remaining fragment becomes a sub-sector.

Conceptually:

```text
Sector Polygon
       │
       ▼
Split By BSP Plane
       │
 ┌─────┴─────┐
 ▼           ▼
Front      Back
       ...
       ▼
     Leaf
```

This produces convex regions that correspond to BSP leaves.

---

# Sector Lookup

Player sector detection does not require polygon testing.

The BSP is traversed exactly like a binary search:

```cpp
while (!leaf)
{
    if (PointSide(splitter, playerPos) >= 0)
        go front;
    else
        go back;
}
```

Once a leaf node is reached, its stored sub-sector immediately identifies the containing sector.

This provides near O(log n) lookup performance on a reasonably balanced tree.

---

# Rendering Traversal

The BSP can be traversed in painter's order using the player's position.

When the player is in front of a splitter:

```text
Render Back
Render Splitter
Render Front
```

When behind:

```text
Render Front
Render Splitter
Render Back
```

This naturally produces a back-to-front wall ordering.

```cpp
CollectBSPWalls(...)
```

returns walls in visibility order without requiring a depth buffer.

# Features

* Automatic BSP generation from wall geometry
* Splitter scoring heuristic
* Dynamic wall splitting
* Convex BSP leaf generation
* Sector-to-leaf mapping
* Fast player sector lookup
* Painter's-order traversal
* Portal-connected sectors
* Variable floor and ceiling heights
* Software-rendered 2.5D world
